# Техническое задание

## 1. Назначение программы

Программа реализует игру "Жизнь" Джона Конвея.
Она моделирует клеточный автомат, где состояние ячеек двумерного поля меняется по правилам Конвея.
Начальное состояние поля задается пользователем.

Задание начального состояния возможно следующими способами:
- Через файл (./game < asset.txt)
- Вручную через терминал после запуска программы

Вывод поля происходит непрерывно с заданным интервалом - раз в определенный промежуток времени, определяемый пользователем.
Каждый шаг моделирования знаменует новое поколение клеток.

## 2. Функциональные требования
### 2.1 Работа с полем
    Программа вычисляет следующее состояние поля на основе текущего.
    На каждом шаге моделирования она просчитывает следующее состояние поля на основе состояния текущего по правилам игры.
### 2.2 Правила игры
    Программа работает по следующим правилам:
    - Если у живой клетки меньше 2 живых соседей, то она погибает от одиночества.
    - Если у живой клетки больше 3 живых соседей, то она погибает от перенаселения.
    - Если у живой клетки 2 или 3 соседа, то она остается живой.
    - Если у мертвой клетки 3 живых соседа, то она оживает.
### 2.3 Ввод
    Программа поддерживает 2 способа ввода:
    - Ручной ввод. Ввод начального состояния поля в консоль(O - живая клетка, все остальное - мертвая клетка)
    - Перенаправленный ввод(./game < asset.txt). 
    Программа принимает на ввод любые символа, все кроме O читаются как мертвая клетка. 
    В случае, если вводимая строка длиннее ширины поля, часть строки, равная ширине, возьмется как текущая строка поля, оставшаяся как следующая.
    В случае, если вводимая строка короче ширины поля, оставшаяся часть дозаполниться мертвыми клетками.
    В случае, если ввод закончился раньше, чем заполнилось поле, оставшаяся часть заполниться мертвыми клетками.
### 2.4 Вывод
    Программа выводит текущее состояние поля, в котором O означает живую клетку, пробелы мертвую.
    Под нижней границей поля выводится управление, а также текущий интервал обновления поля.
### 2.5 Управление
    Программа предоставляет следующие возможности управления:
    - Изменение промежутка обновления поля: a или A - уменьшить(ускорить) , z или Z - увеличить(замедлить)
    - Завершение игры: пробел - для выхода из программы.

## 3. Нефункциональные требования

Программа работает в терминале, не требует графического интерфейса, не зависит от платформы, корректно обрабатывает ввод.

## 4. Формат входных данных

Программа не принимает аргументы командной строки.
Программа обрабатывает ввод, включая перенаправленный с файла.
Файл может содержать в себе любые символы, все кроме O считается мертвой клеткой.

## 5. Тестирование

Тестирование программы включает в себя:
- Юнит тесты логики исполнения игры.
- Проверка работоспособности программы на шаблонах поля.
